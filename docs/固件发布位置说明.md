# 固件发布位置说明

## 发布入口

固件通过 GitHub Actions 编译并发布到阿里云 OSS，当前绑定域名为：

```text
https://firmware.undersilicon.cn
```

最新版本清单地址：

```text
https://firmware.undersilicon.cn/firmware/latest/version.json
```
注意 访问清单时应该增加一个随机的query参数 防止被缓存影响

HTTP 访问会自动跳转到 HTTPS。

## 版本清单

`version.json` 是开发者和服务端判断最新固件版本的入口，包含：

- `latest_version`: 最新版本号，对应 Git tag
- `generated_at`: 清单生成时间
- `commit`: 发布对应的 Git commit
- `firmwares`: 所有固件文件列表

每个固件条目包含：

- `device`: 设备类型
- `kind`: 固件类型，`app` 为 OTA 应用镜像，`merged` 为完整烧录镜像
- `filename`: 文件名
- `object_key`: OSS 对象路径
- `size_bytes`: 文件大小
- `sha256`: 文件 SHA256

## 固件下载规则

固件统一发布在：

```text
https://firmware.undersilicon.cn/firmware/latest/
```

文件名包含设备类型和版本号，例如：

```text
under_silicon_CUNZHI01_v1.1.33.bin
under_silicon_CUNZHI01_v1.1.33_merged.bin
```

OTA 升级应使用 `kind=app` 的应用镜像：

```text
https://firmware.undersilicon.cn/{object_key}
```

完整烧录应使用 `kind=merged` 的合并镜像。

## 发布流程

发布新版本时：

1. 修改 `CMakeLists.txt` 中的 `PROJECT_VER`
2. 提交代码
3. 创建并推送同名 Git tag，例如 `v1.1.34`
4. GitHub Actions 自动构建 Release，并上传固件和 `version.json` 到 OSS

OSS 上传失败不会影响 GitHub Release 的正常创建。
